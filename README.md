# AIC-1102 Lab Workspace

This repository is a hands-on collection of AI-assisted engineering labs used for guided practice, troubleshooting, and workflow improvement across multiple scenarios (debugging, CI/CD, scaffolding, security, MCP tooling, and capstone work).

## What This Project Is

- A structured lab set for building practical AI-assisted engineering habits.
- A workspace with parallel `lab/` and `solutions/` tracks in most modules.
- A place to practice reducing repetitive engineering toil while preserving quality and accountability.

## What We Have Done So Far (High Level)

- Established and aligned custom reusable skills under `.github/skills/`:
  - `lab-error-responder`
  - `lesson-accelerator`
- Standardized lesson guidance workflow around the course guide and intended-observation checks.
- Generated and maintained scaffold-style learner notes across multiple labs to reduce clerical overhead.
- Added a `learnings/` area with trainer-focused reference material for train-the-trainer use.

## Repository Structure (Top Level)

- Lab modules (for example: `guided-multi-step-coding-task/`, `custom-configuration/`, `mcp-tool-design/`, `cicd-integration/`, `capstone/`)
- Course guide: `AIC-1102-LabGuide.html`
- Shared skills and agent-support configuration: `.github/`
- Reusable enablement artifacts: `learnings/`

## How To Use This Workspace

1. Pick a module folder that matches your current learning objective.
2. Start with that module's `lab/README.md`.
3. Use `solutions/` only for comparison or validation after attempting the work.
4. Capture outcomes in the module's expected notes artifact.

## Working Principle

Treat AI as a force multiplier:

- Automate repeatable toil.
- Keep humans accountable for decisions, risk, and acceptance criteria.
- Document what worked so teams can reuse proven patterns.
